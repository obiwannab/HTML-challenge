What is the challange?

This page has been created using some of the most common HTML and CSS errors I have seen, and some of the worst practices.

This page uses out-dated code practices that are no longer considered best practice, and it definitely doesn't look like the mock-up provided.

Overview of your goals

You need to get this page up and running.

  1. Remove tags that have been removed from the HTML5 standard. See this (http://www.w3schools.com/tags/default.asp) handy list from W3Schools for what tags have been removed from HTML5. Using the new semantic tags like <article>, <section>, <aside>, <header>, and <footer> would be ideal, but you may also use <div> tags if you prefer.
  2. Update to best styling practices. Remove all inline styles used and create / link an external style sheet for all necessary CSS.
  3. Make it responsive. Most sites are viewed on mobile devices now, so we need to create websites that have been optimized for viewing on mobile as well as desktop. I would suggest using @media queries, but you can do it anyway you like.
  4. Does it validate?Make sure your page will validate using W3C Markup Validation Service(https://validator.w3.org/)

Want to do more?

Getting the page layout to match the design mock-up and getting the code to validate is your main goals on this Code Challange. If you finish it and want to expand on this project, here are some ideas:

  • Use a framework like Foundation(http://foundation.zurb.com/) or Bootstrap(http://getbootstrap.com/) to build the site out.
  • Create a matching "About" page.
  • Create a matching "Contact" page with a form that users can use to send a quick message, as well as links to connect on social media like LinkedIn or Facebook.
  • Create a matching "Gallery" page to display pictures.
  • If you have already gotten to the JavaScrpt and jQuery courses, you could integrate your image slider into the hero image on this homepage, or turn the side navigation into a drop-down menu.

Final Page Requirements:

  • Navigation and Links
    • Navigation links must be clickable, but they don't need to actually go anywhere
    • Links that go to other outside websites should open in a new tab or window
  • Typography
    • Heading fonts should be in the Rammetto One font. (Use google fonts) If you really don't like this font, feel free to pick a different font that you like better, but it must be a webfont and the font for the headings must be different from the font used for the rest of the site.
  • Images
    • The placeholder image must stretch to sit over the area above the main text of the site, but nbe above the navigation link.
    • The sea image needs to stay stuck at the bottom of the screen, whatever size screen it is being viewed on.
  • Validation and Best Practices
    • Your site either needs to pass validation, or you need a good reason / explination for why you are leaving certain "errors" present.
    • Follow good SEO practices like adding alt text to images.
    • CSS should be in an attached external style sheet, and should be well organized.
    • Use comments as needed
